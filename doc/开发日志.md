# 开发日志

## 2026-03-26

### 训练数据生成器 — 第一步：Commands 校验器

**目标：** 开发训练数据生成器系统（Python 后端 + Vue 前端，文件存储），首先完成 commands 主表的校验功能。

**完成内容：**

1. **设计文档确认**
   - 阅读 `doc/指令主表生成提示词.md`，明确 command registry 的结构规范与约束规则

2. **MMORPG 场景 Commands 主表**
   - 文件：`commands/mmorpg.json`
   - 共 20 条指令，涵盖攻击、跟随、驻守、移动、后撤、防守、掩护、治疗、复活、增益、嘲讽、打断、控制、驱散、施放技能、使用道具等
   - 参数分布：无参数 4 个 / 仅 target 13 个 / 仅 use 2 个 / use+target 1 个

3. **Commands 校验器**
   - 文件：`scripts/validate_commands.py`
   - 校验规则（基于设计文档）：
     - JSON 格式与根结构合法性
     - `command_id`：全大写+下划线、无 `CMD_` 前缀、无重复
     - `slots`：name 只允许 `use`/`target`、type 只允许 `use_span`/`target_span`、name-type 一致性、禁止 `multi` 字段、最多 2 个、双参数必须是 use+target
     - `aliases`：占位符与 slots 定义一致、数量范围 3-6
     - command 总数范围 15-25
   - 额外输出：
     - 无参数/有参数数量对比
     - 单参数/双参数数量对比
     - 逐条文本打印 command_id、desc、slots、aliases

4. **工具使用文档**
   - 文件：`doc/tool.md`
   - 包含校验器的使用方式、校验规则表、输出说明、JSON 格式参考

5. **LLM 配置**
   - 文件：`LLM.txt`
   - 配置 API 地址、密钥及可用模型列表（glm-5, kimi-k2.5, mimo-v2-pro, deepseek-v3.2）

### 训练数据生成器 — 模板主导 + 扰动扩写

**目标：** 基于 commands 主表，使用 LLM 自动生成三类训练数据（模板样本 60-70%、最小编辑对抗样本 20-30%、自由 paraphrase ~10%）。

**完成内容：**

1. **LLM 客户端**
   - 文件：`scripts/llm_client.py`
   - 封装 API 调用（读取 `LLM.txt` 配置）、重试机制、JSON 解析

2. **填槽词库生成**
   - 文件：`scripts/generate_vocab.py`、`scripts/prompts/vocab_prompt.txt`
   - 使用 LLM 生成 MMORPG 背景的 target/use 词库及合理配对
   - 输出：`output/vocab/mmorpg_vocab.json`（54 targets、32 uses、40 配对）

3. **别名模板扩写**
   - 文件：`scripts/expand_aliases.py`、`scripts/prompts/alias_expand_prompt.txt`
   - 使用 LLM 对每个 command 的 aliases 扩写 10-15 条新口语化模板
   - 保持槽位占位符不变
   - 输出：`output/expanded_aliases/mmorpg_aliases*.json`

4. **程序模板样本生成**
   - 文件：`scripts/generate_template_samples.py`
   - 扩写后的别名模板 × 词库进行填槽，生成 label=`quick_command` 正样本
   - 支持双参数配对填充、去重

5. **最小编辑对抗样本**
   - 文件：`scripts/generate_adversarial_samples.py`、`scripts/prompts/adversarial_prompt.txt`
   - 从正样本做 4 种最小编辑：加条件→tactical、加多步→tactical、改模糊→tactical、改闲聊→chat
   - 分批调用 LLM（每批 10 条）

6. **自由 Paraphrase 样本**
   - 文件：`scripts/generate_paraphrase_samples.py`、`scripts/prompts/paraphrase_prompt.txt`
   - 对少量正样本自由改写，每条生成 2-3 个变体

7. **主入口脚本**
   - 文件：`scripts/generate_training_data.py`
   - 串联全流程：词库→扩写→模板→对抗→paraphrase→合并
   - 支持 `--command_id` 单命令测试、`--skip_vocab`/`--skip_aliases` 跳过已有步骤

8. **CAST_ON_TARGET 端到端测试通过**
   - 词库：54 targets、32 uses、40 配对
   - 别名：5 → 19 条模板
   - 生成数据：40 模板 + 40 对抗（30 tactical + 10 chat） + 15 paraphrase = 95 条
   - 数据分布：quick_command 57.9%、tactical 31.6%、chat 10.5%

9. **输出目录重构**
   - 将分散的 `output/vocab/`、`output/expanded_aliases/`、`output/training_data/` 统一为按 command 分目录结构
   - 新结构：`output/{game}/{COMMAND_ID}/`，每个 command 的所有产出（aliases、template、adversarial、paraphrase、merged）集中管理
   - 共享数据（vocab、merged_all）放在 `output/{game}/` 下
   - 修改了 6 个脚本的输出/加载路径，测试通过
