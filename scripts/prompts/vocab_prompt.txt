## 任务
请只为“当前这个 command”生成词库，不要生成其他命令会用到的无关词。

输出字段固定为：
- `targets`: 可填入 `{target}` 的完整自然短语
- `uses`: 可填入 `{use}` 的完整自然短语
- `target_use_pairs`: 合理的 target-use 组合，仅在该 command 同时需要 `target` 和 `use` 时填写

## 生成要求
1. 必须严格服从命令语义画像，不要根据你熟悉的某个固定游戏类型自行套模板。
2. 如果这个 command 更偏向地点、建筑、地标、设施、交互装置、掉落物、任务物品，就优先生成这些词。
3. 如果这个 command 更偏向角色/敌人/队友，就优先生成角色或单位词，不要混入无关建筑或材料。
4. `target` 应该是玩家会直接说出口的完整目标短语，可以是角色、单位、建筑物、房间区域、交互物、掉落物、尸体、设施、地标等。
5. `use` 应该是玩家会直接说出口的完整使用对象短语，可以是技能、法术、武器、武器模式、药剂、卷轴、投掷物、工具、任务道具等。
6. 建筑/场景类 target 要覆盖广义场景目标，例如：建筑、楼体、营地、塔楼、据点、入口、桥头、房间、控制台、祭坛、传送门、补给站、炮台、水晶、终端、开关、箱子、矿点、草药点、尸体、掉落物、装备、材料、任务物品。
7. 游戏内物品要区分用途：
   - 可拾取/交互目标可以是装备、材料、任务物品、掉落物、补给物。
   - 可使用的 `use` 只应包含当前 command 合理会“使用”的技能、消耗品、工具、武器或模式。
8. 不要输出泛泛的分类词，例如“一个建筑”“一个物品”“某个技能”；尽量具体自然。
9. 不要生成明显错误的配对：
   - 不要把拾取/交互命令配成战斗技能组合
   - 不要把纯移动命令配成 use 词库
   - 不要把敌对技能配给明显的战利品或建筑，除非该 command 语义明确允许
10. 当命令只需要 `target` 时，`uses` 和 `target_use_pairs` 返回空数组即可。
11. 当命令只需要 `use` 时，`targets` 和 `target_use_pairs` 返回空数组即可。
12. 数量控制：
   - 需要 `target` 时，通常生成 24-60 个高质量 `targets`；若语义非常窄，可以更少。
   - 需要 `use` 时，通常生成 16-40 个高质量 `uses`；若语义非常窄，可以更少。
   - 当命令同时需要 `target` 和 `use` 时，`target_use_pairs` 请生成 18-30 对高质量组合；若命令语义很窄，数量可以更少，但必须合理。
13. `targets` / `uses` 内部不要重复；`target_use_pairs` 也不要重复。
